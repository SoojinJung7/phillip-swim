# 📩 카카오 알림톡 연동 가이드 (Solapi)

필립 수영은 두 순간에 카카오 알림톡을 자동 발송합니다.

1. **예약 확정** — 회원이 강습을 예약해 자리가 확정될 때
2. **대기 자리 안내** — 운영자가 "다음 대기자 호출"을 누를 때

> 코드는 이미 다 연결돼 있습니다. 아래 세팅을 마치고 **Vercel 환경변수 6개**만 넣으면 자동으로 발송됩니다.
> 환경변수가 없으면 발송을 조용히 생략하므로, 세팅 전에도 예약·대기 기능은 그대로 동작합니다.

---

## 1단계 — Solapi 가입 & API 키 발급

1. [solapi.com](https://solapi.com) 가입 (월정액 없음, 건당 결제 · 알림톡 약 9원/건)
2. **콘솔 → 개발/연동 → API Key 관리** 에서 **API Key** 와 **API Secret** 발급
   - 이 두 값이 `SOLAPI_API_KEY`, `SOLAPI_API_SECRET` 입니다.
3. 발송 비용 충전(포인트) — 소액이라도 미리 충전해 두세요.

## 2단계 — 발신번호 등록

- **콘솔 → 발신번호 관리** 에서 필립 수영 대표번호를 등록(ARS 인증).
- 이 번호가 `SOLAPI_SENDER_PHONE` 입니다. (알림톡 실패 시 SMS 대체발송에도 쓰임)

## 3단계 — 카카오 채널 연동(발신프로필)

> 이미 운영 중인 필립 수영 카카오톡 채널을 그대로 연결하면 됩니다.

1. **콘솔 → 카카오 → 채널 연동** 에서 채널을 추가합니다.
   - 필요한 것: **카카오톡 채널 검색용 ID**(@로 시작), **채널 관리자 휴대폰 인증**.
2. 연동이 끝나면 **발신프로필 ID(pfId)** 가 생깁니다.
   - 이 값이 `SOLAPI_KAKAO_PFID` 입니다.

## 4단계 — 알림톡 템플릿 2개 등록 & 승인

**콘솔 → 카카오 → 알림톡 템플릿** 에서 아래 2개를 **그대로** 등록하세요.
심사는 보통 **1~2 영업일** 걸립니다. (예약/대기 안내는 광고가 아니라 대체로 승인됩니다.)

> ⚠️ 변수 토큰 `#{이름}` `#{강습명}` `#{일시}` 은 **철자·형식 그대로** 넣어야 코드와 맞물립니다.

### 템플릿 ① 예약 확정 안내

```
[필립 수영] 예약 확정 안내

#{이름}님, 예약이 확정되었습니다.

▶ 강습: #{강습명}
▶ 일시: #{일시}

시간에 맞춰 방문해 주세요. 🏊
```

- 승인되면 발급되는 **템플릿 ID** → `SOLAPI_TEMPLATE_RESERVATION_CONFIRMED`

### 템플릿 ② 대기 자리 안내

```
[필립 수영] 대기 자리 안내

#{이름}님, 신청하신 강습에 자리가 나서 예약이 가능합니다.

▶ 강습: #{강습명}
▶ 일시: #{일시}

예약 페이지에서 확정해 주세요.
```

- 승인되면 발급되는 **템플릿 ID** → `SOLAPI_TEMPLATE_WAITLIST_CALLED`

## 5단계 — Vercel 환경변수 6개 등록

Vercel 프로젝트 `phillip-swim` → **Settings → Environment Variables** 에 **Production** 으로 추가
(또는 로컬에서 `vercel env add <이름> production`):

| 환경변수 | 값 | 출처 |
|---|---|---|
| `SOLAPI_API_KEY` | API 키 | 1단계 |
| `SOLAPI_API_SECRET` | API 시크릿 | 1단계 |
| `SOLAPI_SENDER_PHONE` | 등록 발신번호 (예: 0212345678) | 2단계 |
| `SOLAPI_KAKAO_PFID` | 발신프로필 ID | 3단계 |
| `SOLAPI_TEMPLATE_RESERVATION_CONFIRMED` | 예약확정 템플릿 ID | 4단계 ① |
| `SOLAPI_TEMPLATE_WAITLIST_CALLED` | 대기안내 템플릿 ID | 4단계 ② |

> 모두 **서버 전용**입니다. 절대 `NEXT_PUBLIC_` 접두사를 붙이지 마세요(브라우저 노출 금지).

등록 후 재배포(빈 커밋 push 또는 Vercel에서 Redeploy)하면 바로 알림톡이 나갑니다.

## 동작 방식 요약 (코드)

- `lib/notify.ts` — 발송 로직. `isAlimtalkConfigured()`로 키 유무를 확인하고, 없으면 로그만 남기고 생략.
- `app/api/reservations/route.ts` — 예약 성공 시 `notifyReservationConfirmed()` 호출.
- `app/admin/actions.ts` — 대기자 호출 시 `notifyWaitlistCalled()` 호출.
- 모든 발송은 **best-effort**: 알림톡이 실패해도 예약·대기 처리는 정상 완료됩니다.
  발신번호가 있으면 알림톡 실패 시 **SMS로 자동 대체발송**합니다.

## 비용 (참고)

- 알림톡: 약 8~9원/건, SMS 대체발송: 약 20원/건.
- 예약·대기 발생 건수만큼만 나가므로 소규모 스튜디오는 월 몇천 원 수준입니다.
