# 🏊 필립 수영 (phillip-swim)

필립 수영 스튜디오를 위한 **수영 강습 예약 + 대기(웨이팅) 웹앱**입니다.
[캐치테이블](https://www.catchtable.co.kr/)의 두 가지 핵심 경험을 수영장에 맞게 벤치마킹했습니다.

- **기능 1 — 예약**: 강습마다 정원이 정해져 있고, 빈자리가 있으면 즉시 예약이 확정됩니다.
- **기능 2 — 대기**: 자리가 다 차면 대기열에 등록해 내 순번을 확인하고, 취소 자리가 나면 운영자가 순서대로 "다음 대기자 호출"을 할 수 있습니다.

휴대폰에서 편하게 쓰도록 **모바일 우선(mobile-first)** 으로 만들었고, 시원한 물빛 블루·틸 색으로 수영 테마를 살렸습니다. 모든 화면은 한국어입니다.

---

## ✨ 주요 기능

### 회원(일반 이용자)
- **간편 로그인**: 비밀번호 없이 이름 + 전화번호만 입력하면 됩니다. 같은 번호로 다시 오면 예약·대기 내역을 그대로 이어서 볼 수 있어요.
- **강습 둘러보기 & 예약** (`/classes`): 다가오는 강습을 날짜순으로 보고, 남은 자리·마감 임박 상태를 색으로 확인한 뒤 바로 예약합니다.
- **대기 등록** (`/classes/[id]/waitlist`): 마감된 강습에 대기를 걸면 "현재 N번째"가 실시간으로 보이고, 자리가 나서 호출되면 "지금 입장하세요 🏊" 안내가 뜹니다.
- **내 예약·대기** (`/mypage`, `/waitlist`): 내가 예약한 강습과 대기 현황을 한 곳에서 확인·취소합니다.

### 운영자(관리자)
- **관리자 콘솔** (`/admin`): 관리자 전화번호로 로그인하면 자동으로 권한이 켜집니다.
- **강습 개설·수정·삭제**: 정원·시간·강사·설명 등을 관리합니다.
- **명단 관리** (`/admin/classes/[id]`): 예약자·대기자 명단을 보고, 예약을 취소하거나 **"다음 대기자 호출"** 버튼으로 대기 순번 1번을 호출합니다.

## 🧱 기술 스택

- **Next.js 14** (App Router) + **TypeScript**
- **Tailwind CSS** (물 테마 커스텀 팔레트)
- **Supabase** (`@supabase/supabase-js`) — 선택 사항
- 세션: 외부 인증 제공자 없이 **httpOnly 쿠키** 기반 경량 세션

> **환경변수가 없으면 인메모리(mock) 저장소로 동작합니다.** 덕분에 별도 설정 없이 `npm run dev`만 실행해도 6개의 샘플 강습이 채워진 완성된 앱을 바로 볼 수 있어요.

## 🚀 로컬에서 실행하기 (설정 0단계, 바로 실행)

```bash
npm install
npm run dev
```

브라우저에서 <http://localhost:3000> 을 엽니다.

- 환경변수를 아무것도 넣지 않아도 됩니다. 샘플 강습 6개가 담긴 **mock 저장소**가 앱 전체를 실제처럼 구동합니다.
- 예약·대기·취소·관리자 기능이 모두 메모리 안에서 실제로 동작합니다. (서버를 재시작하면 초기 샘플 상태로 돌아갑니다.)

### 관리자로 들어가려면

관리자 권한은 로그인한 **전화번호**로 결정됩니다.

- 기본 관리자 번호는 `010-0000-0000` 입니다. 이 번호로 로그인하면 상단에 **관리자** 배지가 뜨고 `/admin` 에 접근할 수 있어요.
- 다른 번호를 관리자로 쓰려면 `ADMIN_PHONE` 환경변수를 설정하세요 (아래 참고).

## 🗄️ Supabase 연결하기 (실데이터로 전환, 선택)

프로토타입을 넘어 데이터를 영구 저장하려면 Supabase를 붙이면 됩니다.

1. [supabase.com](https://supabase.com) 에서 프로젝트를 만듭니다.
2. `.env.example` 을 복사해 `.env.local` 을 만들고 키를 채웁니다.

   ```bash
   cp .env.example .env.local
   ```

   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   # 관리자 전화번호 (이 번호로 로그인하면 관리자 권한이 켜집니다)
   ADMIN_PHONE=010-1234-5678
   ```

   > URL과 anon key는 Supabase 대시보드 → **Project Settings → API** 에서 확인합니다.

3. Supabase 대시보드의 **SQL Editor** 에 `supabase/schema.sql` 전체를 붙여넣고 실행합니다.
   테이블(`members`, `swim_classes`, `reservations`, `waitlist`)과 인덱스, 그리고 mock과 동일한 샘플 강습 6개가 생성됩니다.

4. 개발 서버를 재시작합니다.

   ```bash
   npm run dev
   ```

   `NEXT_PUBLIC_SUPABASE_URL` 과 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 가 모두 있으면 앱이 자동으로 Supabase 모드로 전환됩니다. (둘 중 하나라도 없으면 mock으로 동작합니다.)

### ⚠️ ADMIN_PHONE 참고

- `ADMIN_PHONE` 을 설정하지 않으면 기본값 `010-0000-0000` 이 관리자 번호가 됩니다.
- 전화번호는 숫자만 비교하므로 `010-1234-5678` 과 `01012345678` 은 같은 번호로 취급됩니다.
- 관리자 여부는 로그인할 때가 아니라 **매 요청마다 최신 `ADMIN_PHONE` 기준**으로 다시 계산됩니다. 배포 후 관리자 번호를 바꿔도 즉시 반영됩니다.
- `schema.sql` 에는 RLS(Row Level Security)가 기본 비활성으로 되어 있습니다. **실서비스 전에는 파일 하단의 예시를 참고해 RLS 정책을 반드시 추가**하세요.

## 🗺️ 페이지(라우트) 지도

| 경로 | 설명 | 대상 |
| --- | --- | --- |
| `/` | 홈 — 히어로, 이용 안내(예약/대기), 다가오는 강습 미리보기 | 모두 |
| `/classes` | 강습 목록 — 남은 자리·마감 상태와 함께 | 모두 |
| `/classes/[id]` | 강습 상세 + 예약 패널 | 모두 |
| `/classes/[id]/waitlist` | 대기 등록 + 실시간 내 순번 | 모두 |
| `/waitlist` | 내 대기 현황 (여러 강습 통합) | 로그인 |
| `/mypage` | 내 예약·대기 요약, 예약 취소, 로그아웃 | 로그인 |
| `/login` | 이름 + 전화번호 간편 로그인 | 모두 |
| `/admin` | 관리자 대시보드 — 강습별 예약/대기 현황 | 관리자 |
| `/admin/classes/new` | 강습 개설 | 관리자 |
| `/admin/classes/[id]` | 예약자·대기자 명단, 다음 대기자 호출 | 관리자 |
| `/admin/classes/[id]/edit` | 강습 수정·삭제 | 관리자 |
| `/api/me` | 현재 로그인 상태 조회(내비 표시용) | 내부 |
| `/api/reservations` | 예약 생성 API | 내부 |

## 📁 프로젝트 구조

```
app/            페이지 & 라우트 핸들러 (App Router)
components/
  ui/           공용 UI (Button, Card, Badge, PageHeader …)
  layout/       상단 내비게이션(SiteNav)
  booking/      예약 화면 컴포넌트
  waitlist/     대기 화면 컴포넌트
  admin/        관리자 화면 컴포넌트
  auth/         로그인/로그아웃/로그인 상태 표시
lib/
  types.ts      공유 타입
  data.ts       데이터 접근 계층 (Supabase ↔ mock 자동 전환)
  auth.ts       쿠키 기반 경량 세션
  supabase/     Supabase 클라이언트 팩토리
supabase/
  schema.sql    Supabase 테이블·인덱스·샘플 데이터
```
